1. Executive Summary 

Climatefacts.ai (formerly CliLens.AI ; rebrand finalised 2026-05-17, commit 1b96d3a ) is a production-grade,
domain-driven, full-stack climate intelligence platform. It is owned and stewarded by CISU Regen ,
whose mission is to “develop regenerative and resilient economies, organisations, and communities of the future.”
The platform’s tagline — “Trust as a Service” — reflects its core promise: every claim ships with
evidence trails, transparent prompts, calibrated confidence, and per-LLM-call provenance. 

The platform has progressed through five visible phases of work since early 2026: 

Phase 1 — Resolve known issues (mock-data purge, schema fixes, Vitest baseline). Complete. 
Phase 2 — Architecture decision (Kafka removed; Celery + Redis adopted; 7-container compose). Complete. 
Phase 3 — EU expansion and global RSS ingestion (215 feeds, 8 regions, 198 reference countries). Complete. 
Phase 4 — “Truth-machine” uplift: provenance, calibration, multi-LLM verification, drift, public methodology, compliance docs. Complete. 
Phase 5 — Cloud cut-over on GCP project climatenews-495412 (region europe-west4 ). In progress. 

The 2026-05-16 → 2026-05-17 push delivered 32 commits that lifted the platform’s self-reported
“truth-machine” composite from 2.2/5 (pre-audit) to a claimed ~4.78/5 .
A subsequent independent analytical audit (2026-05-18, see docs/audits/2026-05-18-analytical-audit.md )
reduces the honestly-defensible composite to ~3.6/5 with concrete, actionable gaps
(see §7 and §10). Both numbers are surfaced in this report to keep the picture truthful for auditors. 

Headline state (2026-05-18). Climatefacts.ai is functionally a complete climate-news verification platform —
ingestion, search, map, deep-search, URL analysis, agentic chat, dashboards, subscriptions, OAuth, methodology surfaces and
admin tooling all operate end-to-end. Six external indicator adapters (Climate TRACE, OWID, Climate Action Tracker, UNFCCC NDC,
IRENA, ND-GAIN) feed the sustainability score. Two pre-existing data-integrity gaps surfaced in the latest enrichment work
(3,588 synthetic seed articles — now flagged via migration 025; and RSS ingestion currently persisting summaries instead of
full article bodies) are the most material credibility risks before launch.

2. Solution Vision 

2.1 Mission statement 
Climatefacts.ai is a global climate communication and truth-verification platform. Anyone, anywhere, in any language,
can obtain fact-checked climate news from their region, accompanied by transparent credibility scoring, country-level
sustainability context, and (in the planned final state) ready-to-share short-form video produced from the same evidence. 

2.2 Strategic positioning 

From localised news service → global communication platform. The original MVP framed itself as a
Finnish climate-news site; the current architecture, feeds and data adapters span 87+ ISO countries today and 198
country surfaces in the UI (with depth growing as ingestion runs land). 
From article aggregator → trust infrastructure. The platform deliberately operates as a
“truth machine”: every analytical output (claim verdict, sustainability score, chat answer, deep-search synthesis)
records its prompt, its model, its retrieval strategy, its source IDs, and its hallucination risk score in the
claim_provenance table, and exposes them at /api/methodology/audit-trail/… and on the public
/methodology page. 
From single product → partner-ready platform. The methodology surface, OAuth (Google + Microsoft),
subscription tiers, API keys and rate-limited endpoints are all built so the same data and models can be embedded
in partner workflows. 

2.3 Audience and tiers 

Audience Use case Subscription tier 
Citizens & journalists Read fact-checked climate news; check a URL; explore country views. Freemium 
Researchers & analysts Deep-search, methodology drawer, sustainability comparisons, audit trails. Basic / Professional

2.4 Long-horizon vision (preserved from docs/VISION_GLOBAL_CLIMATE_PLATFORM.md ) 
The roadmap envisions an end-to-end production pipeline where users can pick any country on a world map and instantly
receive (i) localised, translated, fact-checked articles, (ii) interactive credibility & sustainability dashboards, and
(iii) auto-produced 15–60 s vertical videos for TikTok / Reels / YouTube Shorts. The current build delivers (i) and (ii)
in depth; (iii) remains a planned future phase (Video Production Pipeline — see §8).

5. Agentic Features & AI Pipeline 

5.1 LLM stack 

Role Provider / model Notes 
Primary LLM (claim extraction, chat synthesis, deep search) DeepSeek ( deepseek-reasoner by default) Pinned in .env ; cheap-first preference per article_enrichment_service.py . 
Secondary verifier (multi-LLM cross-check) Anthropic ( ANTHROPIC_MODEL env, default Claude) Used by anthropic_claim_extractor.py + multi_llm_verifier.py . 
Embeddings OpenAI text-embedding-ada-002 HNSW index on article_embeddings (migration 019); FTS fallback if absent.

5.2 The chat / contextual assistant 
The global panel ( ContextualAssistant → AgenticAssistant ) is context-aware — the
current view (route, article id, country, compare countries, analysis id, deep-search query) flows into
view_context on every request. The panel dispatches to one of three backends depending on context: 

POST /api/chat — general Q&A; HybridRAG (FTS + vector + knowledge graph) with Cynefin
classification, view-context hydration and a HallucinationDetector grounding pass. 
POST /api/map/query — map-mode; returns country_highlights for the map page state. 
POST /api/articles/{id}/ask — single-article grounded Q&A on the active article only. 

Honest current limit. The chat is conversational only . There is no tool-call / function-call protocol that
lets the LLM act on the platform — it cannot trigger URL analysis, apply filters, bookmark articles, open a country
panel, navigate to /methodology , run a deep search or write calibration labels.
Adding any of those requires a structured actions: [{type, params, label}] payload from the backend and
client-side dispatchers in AgenticAssistant . See §10, item 1 for the recommended action protocol.

5.3 Truth-machine pipeline (per URL analysis) 

Fetch — SSRF-hardened HTTP fetch with redirect re-validation; URL job-token issued for the run. 
Cynefin classification — cynefin_router.py ports the projectcarfcynepic prompt; falls back
to keyword classifier when CARF is offline. 
Claim extraction — DeepSeek primary; Anthropic secondary (env-gated via
CLILENS_MULTI_LLM_VERIFY=1 ). Both use the same versioned prompt
( claim_extraction v1.0) from intelligence/prompts.py to isolate model divergence from
prompt divergence. 
Multi-LLM cross-verification — Jaccard token-set similarity ≥ 0.5 = corroborated; uncorroborated
claims have confidence multiplied by 0.7; agreement_score recorded. 
Hallucination check — entity overlap (0.30) + statistic accuracy (0.30) + LLM-grounding (0.40) composite
against the article body; flagged segments persisted in claim_provenance.raw_metadata . 
Calibration — raw reliability_score , agreement_score and (1− hallucination_score )
are passed through Platt scalers fitted on labelled reviewer data;
calibrated_reliability_score surfaces on the detail view when a fit exists. 
Provenance — one row per LLM-producing step into claim_provenance with model + prompt name
+ version + SHA-256 fingerprint + retrieval strategy + source_article_ids + raw_metadata; auditable through
/api/methodology/audit-trail/url-analysis/{id} . 

5.4 Truth-machine pipeline (per deep-search session) 
The same provenance + hallucination + Cynefin flow is applied to multi-source synthesis. Each session is keyed by
deep_search_session_id (migration 023). Hallucination scoring runs against the union of titles/snippets
returned by HybridRAG; methodology.hallucination_check is surfaced inline to the user. 

5.5 Drift detection 
intelligence/drift_detection.py implements two KL-divergence pipelines: 

Source-mix drift — recent N days vs. baseline window; alerts when reliance on one publisher creeps up. 
Prompt-fingerprint drift — detects silent prompt edits by comparing recent vs. baseline fingerprint
distributions in claim_provenance . 

Thresholds (0.10 / 0.25 / 0.50 nats) are hard-coded constants today. The audit (§7) flags this for per-deployment
calibration. 

5.6 Multi-agent coordination (development-time, not runtime) 
The repository’s CLAUDE.md declares the SPARC methodology and Claude-Flow swarm orchestration patterns
(specification → pseudocode → architecture → refinement → completion) plus 54 named agent roles. This is a
development-workflow convention, not a runtime feature of the deployed application. The runtime system is a
deterministic FastAPI + Celery service; the multi-agent “swarm” is used by Claude Code / Claude-Flow during
authoring of the code, not when Climatefacts.ai serves user requests.

6.2 Indicator catalogue & adapter framework 
Every external country-level indicator flows through a common adapter ABC ( indicators/base.py :

6.3 Sustainability composite (methodology v1) 
The formula in intelligence/sustainability_score.py is a weighted Bayesian-style composite over the
indicators above: 

sustainability_score = 0.40 * normalize_emissions_per_capita_inv(OWID)
+ 0.40 * renewable_share_electricity(OWID)
+ 0.20 * climate_action_tracker_band(CAT)

Re-weighting on missing components is transparent (denominator drops). Confidence bands widen with fewer indicators:
±25 (1), ±15 (2), ±10 (3+). Methodology version is pinned to every output
( sustainability_v1_2026_05 ). 

Known formula gap (analytical audit, 2026-05-18). UNFCCC NDC, IRENA and ND-GAIN are fetched and stored but
not currently consumed by the composite — only Climate TRACE, OWID and CAT enter the score. ND-GAIN adaptation
readiness in particular biases the composite against vulnerable low-emitter nations. Recommended remediation:
add a fourth component (ND-GAIN, weight 0.15) and rebalance to 0.35/0.35/0.15/0.15, bumping
METHODOLOGY_VERSION to sustainability_v2_2026_05 . See §10, item 2.

6.5 Calibration data plane 

calibration_labels(label_id, url_analysis_id, label_truth, labeled_by, label_method, …) —
reviewer ground-truth. 
calibration_fits(signal, a, b, n_labels, brier, ece, fitted_at, …) — per-signal Platt
parameters; one row per refit. 
Signals: reliability_score , agreement_score , hallucination_score (inverted to
“1 − risk” so higher = better). 
Floor: min_labels=5 for a fit; the analytical audit recommends raising to 50 with a
“preview” tag and a UI banner when active fit was learned on too few labels.

6.7 Versioned prompt registry 
intelligence/prompts.py exposes a PromptTemplate registry with version + fingerprint + rationale.
Every LLM caller fetches via get_prompt(name) rather than inlining strings; the methodology block records
prompts_used.<name> for the run and the public /api/methodology/prompts endpoint surfaces the
full registry. This makes “the prompt changed silently” impossible without bumping the version.

countries table holds the 198-country reference catalogue with continent, lat/long, news_sources_available,

Regions: europe, north_america, africa, latin_america, asia, middle_east, global, research (exposed via

6.9 Synthetic-data hygiene 
Two data-integrity issues surfaced during the 2026-05-18 enrichment session and remain only partially mitigated: 

3,588 synthetic seed articles (1,008 from populate_demo_articles.py with example.com URLs;
2,580 from seed_full_global.py with clilens.ai URLs). Flagged via migration 025 and excluded from
batch_enrich and the backfill script — but still visible to users on the map, search and
compare surfaces . Frontend filtering on is_synthetic = FALSE is the recommended next step. 
RSS adapter stores feed summaries, not article bodies. The Celery RSS pipeline ( rss_adapter.py :
_parse_feed ) reads feed.summary , truncates to 500 chars, and writes to
extracted_text . The full-page extractor exists at
src/backend/services/ingestion_service/src/scraper.py:_fetch_article_content but is only wired into an
unused Kafka consumer. Consequence: every downstream consumer of extracted_text (claim extraction,
hallucination check, enrichment, embeddings, search) operates on title-plus-summary stubs. The truth-machine work
is honest about what it does — but its inputs are smaller than expected. Estimated fix: 3–4 hours
(add trafilatura ; fetch+extract on RSS ingest with timeout + fallback).

11. Glossary & Reference 

Term Meaning 
CARF Causal Analytical Reasoning Framework — optional external HTTP reasoning engine; fallback is the Cynefin keyword classifier. 
Cynefin Sense-making framework used to route claims between simple / complicated / complex / chaotic reasoning paths. 
HybridRAG Composed retrieval over FTS + pgvector + knowledge graph, pre-routed by a Cynefin classifier. 
HNSW Hierarchical Navigable Small World — the pgvector ANN index used for embedding similarity. 
Platt scaling Two-parameter logistic regression that maps raw model confidences to calibrated probabilities. 
Brier / ECE Brier score and Expected Calibration Error — the two scalar metrics surfaced for each calibration fit. 
KL divergence Information-theoretic distance between probability distributions; used for drift detection on source mix and prompt fingerprint. 
ND-GAIN Notre Dame Global Adaptation Index — country adaptation readiness score. 
CAT Climate Action Tracker — 5-band per-country policy rating mapped to 0–100. 
OWID Our World in Data — CSV-distributed canonical climate datasets. 
Claim provenance The audit table that records model + prompt + version + fingerprint + retrieval strategy + source IDs + hallucination score for every LLM-producing step. 
Methodology version Pinned identifier (e.g. sustainability_v1_2026_05 ) attached to every analytical output so historical scores remain reproducible after formula changes.
