## Executive summary
Climatefacts.ai is an ambitious, architecturally-rich platform that is NOT yet best-in-world and not yet production-trustworthy. The dominant theme across all 15 audits is "built but not wired/triggered" — sophisticated machinery ships behind a dead or default-off switch. Verified live by me today (2026-06-02): (1) GET /api/v2/articles -> 200 [] and ?q=climate -> 500, while /api/v2/stats reports 3,329 articles — the canonical content API is effectively dead, root-caused to repository.py:228-230 (hardcoded climate-tag overlap filter, default include_non_climate=False) + an f-string to_tsvector over a.headline/summary_text. This single bug is independently flagged by 6 of 15 audits (backend, data-sources, semantic, docs-schema, market, cost). (2) POST /api/chat returns actions:[] even for an explicitly action-eliciting prompt — the headline "chat that acts" surface is dead because _generate_answer never instructs the LLM to emit actions; all downstream plumbing (22-skill registry, dispatcher, confirm modal) is built and idle. (3) /api/v2/intelligence/analyze-text is reachable unauthenticated (422 validation-only, no 401) — an ungated LLM-cost sink that also gives away the exact capability the monetization thesis wants to sell.

Three platform-wide credibility gaps undercut the "best fact-insight platform" mandate: the displayed URL credibility score is an extraction-length heuristic (url_analysis_routes.py:1606-1621, self-commented as such) labelled "Credibility"; embeddings are 0/666 so semantic similarity/RAG is FTS+graph only; and world coverage is ~20 countries (10.4% of UN-193) with Africa=4/LatAm=1 RSS sources. Honesty surfaces (calibration n=0, provenance empty, drift "stable" on zero data) render confident-looking badges over no data, the opposite of the methodology page's purpose. CI runs only ~31% of the backend test corpus and Playwright is not wired in at all, so these regressions ship undetected — test_skills_registry (IN the CI list) asserts 11 skills against a 22-skill reality, meaning CI is red or being bypassed.

Top themes: (A) a handful of S-effort "wire the existing thing" fixes (v2/articles default, chat-actions prompt, gate analyze-text, parameterize SQL, GIT_COMMIT_SHA) unlock disproportionate value; (B) the GX10 free-inference box is the lever for embeddings, translation, and relevance at zero marginal cost; (C) honesty/methodology fixes are mandatory before any API monetization or coverage marketing; (D) coverage and monetization are real but contingent on first proving the engine robust (embeddings populated, sources monitored, scores honest).
